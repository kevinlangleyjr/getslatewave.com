Cursor uses the same VSIX extension format and Microsoft Marketplace
gallery as VSCode, so there's no separate Cursor port — the
[Slatewave for VSCode](/themes/vscode/) extension installs into Cursor
unchanged. Same syntax tokens, same workbench chrome, same diff colors.

The reason Cursor gets its own page (and its own
`slatewave install cursor`) is purely operational: the CLI needs to
shell out to the `cursor` binary instead of `code`, and `slatewave
uninstall cursor` should only reverse the Cursor side if you happen to
have both editors with Slatewave installed.

For the full preview, palette mapping notes, and the editor-themed
screenshots, see [Slatewave for VSCode](/themes/vscode/).
